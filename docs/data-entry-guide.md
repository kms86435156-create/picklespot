# PBL.SYS 운영자 데이터 입력 가이드

> 이 문서는 실제 운영 데이터를 입력하는 담당자를 위한 실무 가이드입니다.
> 입력 순서, 필수/권장 항목, 품질 기준을 다룹니다.

---

## 입력 순서

아래 순서를 지켜야 홈 화면이 자연스럽게 채워집니다.

### Step 1. 대회 (목표: 30건 이상)

대회가 가장 먼저입니다. 홈 메인의 "추천 대회", "이번 달 대회", "마감 임박" 섹션을 채웁니다.

**최소 필수 입력값:**
| 필드 | 예시 | 비워두면 |
|------|------|----------|
| 대회명 | 제1회 서울 오픈 피클볼 대회 | 등록 불가 |
| 시작일 | 2026-05-10 | 이번 달 대회 목록에 안 나옴 |
| 지역 | 서울 | 지역 필터에서 누락 |
| 상태 | open | 홈에서 "접수중"으로 안 뜸 |

**권장 입력값:**
| 필드 | 비워두면 어떻게 보이나 |
|------|----------------------|
| 접수마감일 | "마감 임박" 섹션에 안 나옴 |
| 장소명 | 상세에서 "장소 확인중"으로 표시 |
| 참가비 | ₩0으로 표시 (무료 대회로 오해) |
| 종별 | 상세 페이지에서 빈칸 |
| 정원 | 참가 현황 표시 안 됨 |
| 주최/연락처 | 신뢰감 떨어짐 |
| 설명 | 상세 페이지가 빈약해 보임 |

### Step 2. 피클볼장 (목표: 100건 이상)

피클볼장은 지역별 분포가 중요합니다. 서울/경기 집중보다 전국에 고르게 분포시키세요.

**최소 필수 입력값:**
| 필드 | 예시 | 비워두면 |
|------|------|----------|
| 이름 | 잠실 피클볼 아레나 | 등록 불가 |
| 지역 | 서울 | 지역 필터/지역별 카운트에서 누락 |

**권장 입력값:**
| 필드 | 비워두면 어떻게 보이나 |
|------|----------------------|
| 주소/도로명주소 | 위치 정보 없음 |
| 코트수 | 추천 장소 자동 선정에서 밀림 |
| 실내외 | 카드에서 빈칸 |
| 전화번호 | 연락처 없음으로 표시 |
| 운영시간 | 방문 전 확인 불가 |
| 주차여부 | 누락 |

### Step 3. 동호회 (목표: 20건 이상)

동호회는 "모집중" 상태가 있어야 홈에 노출됩니다.

**최소 필수 입력값:**
| 필드 | 예시 | 비워두면 |
|------|------|----------|
| 동호회명 | 강남피클러스 | 등록 불가 |
| 지역 | 서울 | 지역 필터에서 누락 |

**권장 입력값:**
| 필드 | 비워두면 어떻게 보이나 |
|------|----------------------|
| 회원수 | 규모감 없음 |
| 활동장소 | 어디서 치는지 모름 |
| 가입방법 | 가입 경로 없음 |
| 소개 | 카드에서 설명 영역 빈칸 |
| 담당자/연락처 | 문의 불가 |

### Step 4. Featured 설정 (마지막)

데이터 입력이 끝난 후에 설정합니다.

- `/admin/tournaments` → 접수중이고 마감 7일 이상 남은 대회 **3~5개** ★ 클릭
- `/admin/venues` → 정보가 풍부하고 코트 수 많은 장소 **4~8개** ★ 클릭
- `/admin/clubs` → 모집중이고 소개가 충실한 동호회 **3~4개** ★ 클릭

> Featured가 0개이면 자동 fallback이 동작하지만, 의도적으로 설정하는 것이 품질에 좋습니다.

## 입력 방법

### 방법 1: 관리자 페이지에서 수동 입력

1. `/admin` → 해당 관리 페이지 접속
2. `+ 새로 등록` 클릭
3. 필수 필드 입력 후 `등록하기`
4. **연속 입력 모드**: 하단 체크박스 ON → 등록 후 자동으로 다음 입력 (지역/주최 유지)

### 방법 2: CSV 대량 업로드

1. 관리 페이지에서 `CSV 샘플 다운로드` 클릭
2. 엑셀에서 데이터 입력
3. **반드시 CSV UTF-8 형식으로 저장** (엑셀 → 다른 이름으로 저장 → "CSV UTF-8")
4. 관리 페이지의 점선 영역에 드래그앤드롭
5. 업로드 결과 확인 (성공/오류/중복 건수)

## 입력 품질이 낮으면 이렇게 보입니다

### 홈 화면

| 상황 | 결과 |
|------|------|
| 대회 시작일 없음 | "이번 달 대회" 섹션에서 누락 |
| 접수마감일 없음 | "마감 임박" 섹션에서 누락 |
| 참가비 미입력 | ₩0으로 표시 → 무료 대회로 오해 |
| 장소 지역 없음 | "지역별 피클볼장" 카운트에서 빠짐 |
| Featured 0개 | 자동 fallback (의도와 다른 순서로 표시될 수 있음) |

### 상세 페이지

| 상황 | 결과 |
|------|------|
| 대회 장소명 없음 | "장소 확인중"으로 표시 |
| 대회 종별 없음 | 빈칸 |
| 대회 설명 없음 | 상세가 빈약해 보임 |
| 장소 주소 없음 | 위치 정보 영역 비어있음 |
| 장소 운영시간 없음 | 운영시간 표시 안 됨 |
| 동호회 소개 없음 | 카드 하단 빈칸 |

## 자주 나는 업로드 오류

| 오류 | 원인 | 해결 |
|------|------|------|
| `대회명 필드가 비어있습니다` | 필수 컬럼 누락 | 해당 행에 이름 입력 |
| `XXX 중복` | 같은 이름이 존재 | 이름 변경 또는 기존 수정 |
| `Empty CSV` | 데이터 없음 | 헤더 아래에 행 추가 |
| 글자 깨짐 | UTF-8 아닌 인코딩 | 엑셀에서 "CSV UTF-8"로 저장 |

## Supabase 연결 방법

1. [Supabase](https://supabase.com)에서 프로젝트 생성
2. SQL Editor에서 `supabase/migration.sql` 실행
3. `.env`에 환경변수 설정:
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
   SUPABASE_SERVICE_ROLE_KEY=eyJ...
   ADMIN_USERNAME=admin
   ADMIN_PASSWORD=your-password
   ```
4. 시드 데이터 투입: `npx tsx src/scripts/seed-supabase.ts`
5. 확인: `curl http://localhost:3000/api/health`
